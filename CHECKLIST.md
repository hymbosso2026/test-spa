CHECKLIST POUR LE PROFESSEUR — Vérification rapide

Lien du site : https://hymbosso2026.github.io/test-spa/

But : guider la vérification et faciliter la notation.

Instructions pratiques

1) Ouvrir l’URL du projet dans un navigateur de bureau (Chrome/Firefox/Edge).
2) Vérifier la page d’accueil : titre, menu, accessibilité de la navigation.
3) Tester la création d’un travel note :
   - Si la création est publique : cliquer sur « Nouveau » / « Publish » et remplir le formulaire.
   - Si une authentification est requise : demander un compte test à l’étudiant.
4) Vérifier la galerie : charger une image, voir l’affichage responsive et les captions.
5) Tester la recherche et les filtres : rechercher par mot-clé et filtrer par catégorie.
6) Tester la responsivité : redimensionner la fenêtre ou ouvrir sur mobile.
7) Vérifier la performance basique : navigation fluide, pas d’erreur 500 dans la console réseau.
8) Vérifier la documentation :
   - LIRE `README.md` dans le dépôt GitHub
   - OUVRIR les fichiers `public/demo.html` et `public/ppt.html` si nécessaire
9) Vérifier le rapport final : demander le fichier Word/PDF généré (Training Project Report) si besoin.

Critères de notation recommandés (exemples)

- Fonctionnalité : 40% (création, publication, galerie, recherche)
- Qualité UI / UX : 20% (lisibilité, responsive, styles)
- Documentation et présentation : 20% (README, rapport, clarté)
- Code & architecture : 10% (structure, lisibilité, modularité)
- Originalité / valeur pédagogique : 10%

Remarques pour le professeur

- Si un compte test est nécessaire, contactez l'étudiant (Nkwemi Keyim Hymbosso).
- Pour toute difficulté d’accès, fournir la capture d’écran et l’heure de la vérification.

Merci — Checklist fournie par l’étudiant pour faciliter l’évaluation.
